<p><b>Markdown</b> is a <a href="/wiki/Lightweight_markup_language" title="Lightweight markup language">lightweight markup language</a> with plain text formatting syntax. It is designed so that it can be converted to <a href="/wiki/HTML" title="HTML">HTML</a> and many other formats using a tool by the same name.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> Markdown is often used to format <a href="/wiki/README" title="README">readme files</a>, for writing messages in online discussion forums, and to create <a href="/wiki/Formatted_text" title="Formatted text">rich text</a> using a <a href="/wiki/Plain_text" title="Plain text">plain text</a> <a href="/wiki/Text_editor" title="Text editor">editor</a>. As the initial description of Markdown contained ambiguities and unanswered questions, many implementations and extensions of Markdown appeared over the years to answer these issues.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Markdown&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/John_Gruber" title="John Gruber">John Gruber</a> created the Markdown language in 2004 in collaboration with <a href="/wiki/Aaron_Swartz" title="Aaron Swartz">Aaron Swartz</a> on the syntax,<sup id="cite_ref-markdown-swartz_3-1" class="reference"><a href="#cite_note-markdown-swartz-3">[3]</a></sup><sup id="cite_ref-gruber-2004-release_4-1" class="reference"><a href="#cite_note-gruber-2004-release-4">[4]</a></sup> with the goal of enabling people "to write using an easy-to-read, easy-to-write plain text format, and optionally convert it to structurally valid <a href="/wiki/XHTML" title="XHTML">XHTML</a> (or <a href="/wiki/HTML" title="HTML">HTML</a>)".<sup id="cite_ref-md_5-1" class="reference"><a href="#cite_note-md-5">[5]</a></sup></p>
<p>The key design goal is <i>readability</i> – that the language be readable as-is, without looking like it has been marked up with tags or formatting instructions,<sup id="cite_ref-philosophy_9-0" class="reference"><a href="#cite_note-philosophy-9">[9]</a></sup> unlike text formatted with a <a href="/wiki/Markup_language" title="Markup language">markup language</a>, such as <a href="/wiki/Rich_Text_Format" title="Rich Text Format">Rich Text Format</a> (RTF) or HTML, which have obvious tags and formatting instructions. To this end, its main inspiration is the existing <a href="/wiki/Convention_(norm)" title="Convention (norm)">conventions</a> for marking up <a href="/wiki/Plain_text" title="Plain text">plain text</a> in <a href="/wiki/Email" title="Email">email</a>, though it also draws from earlier markup languages, notably <a href="/wiki/Setext" title="Setext">setext</a>, <a href="/wiki/Textile_(markup_language)" title="Textile (markup language)">Textile</a>, and <a href="/wiki/ReStructuredText" title="ReStructuredText">reStructuredText</a>.</p>
<p>Gruber wrote a <a href="/wiki/Perl" title="Perl">Perl</a> script, <code class="mw-highlight" id="" style="" dir="ltr">Markdown.pl</code>, which converts marked-up text input to valid, <a href="/wiki/XML#Well-formedness_and_error-handling" title="XML">well-formed</a> XHTML or HTML and replaces left-pointing angle brackets '<code class="mw-highlight" id="" style="" dir="ltr">&lt;</code>' and <a href="/wiki/Ampersand" title="Ampersand">ampersands</a> '<code class="mw-highlight" id="" style="" dir="ltr">&amp;</code>' with their corresponding <a href="/wiki/Character_entity_references" class="mw-redirect" title="Character entity references">character entity references</a>. It can be used as a standalone script, as a plugin for <a href="/wiki/Blosxom" title="Blosxom">Blosxom</a> or <a href="/wiki/Movable_Type" title="Movable Type">Movable Type</a>, or as a text filter for <a href="/wiki/BBEdit" title="BBEdit">BBEdit</a>.<sup id="cite_ref-md_5-2" class="reference"><a href="#cite_note-md-5">[5]</a></sup></p>
<p>Markdown has since been re-implemented by others as a Perl <a href="/wiki/Modular_programming" title="Modular programming">module</a> available on <a href="/wiki/CPAN" title="CPAN">CPAN</a> (<code class="mw-highlight" id="" style="" dir="ltr">Text::Markdown</code>), and in a variety of other programming languages.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup> It is distributed under a <a href="/wiki/BSD_license" class="mw-redirect" title="BSD license">BSD-style license</a> and is included with, or available as a plugin for, several <a href="/wiki/Content_management_system" title="Content management system">content-management systems</a>.<sup id="cite_ref-license_7-1" class="reference"><a href="#cite_note-license-7">[7]</a></sup><sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup><sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup></p>
<p>Sites such as <a href="/wiki/GitHub" title="GitHub">GitHub</a>, <a href="/wiki/Bitbucket" title="Bitbucket">Bitbucket</a>, <a href="/wiki/Reddit" title="Reddit">Reddit</a>, <a href="/wiki/Diaspora_(social_network)" title="Diaspora (social network)">Diaspora</a>, <a href="/wiki/Stack_Exchange" title="Stack Exchange">Stack Exchange</a>, <a href="/wiki/OpenStreetMap" title="OpenStreetMap">OpenStreetMap</a>, and <a href="/wiki/SourceForge" title="SourceForge">SourceForge</a> use variants of Markdown to facilitate discussion between users.<sup id="cite_ref-gfm_on_github_13-0" class="reference"><a href="#cite_note-gfm_on_github-13">[13]</a></sup><sup id="cite_ref-14" class="reference"><a href="#cite_note-14">[14]</a></sup><sup id="cite_ref-15" class="reference"><a href="#cite_note-15">[15]</a></sup><sup id="cite_ref-16" class="reference"><a href="#cite_note-16">[16]</a></sup></p>
<h3><span class="mw-headline" id="Standardization">Standardization</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Markdown&amp;action=edit&amp;section=2" title="Edit section: Standardization">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>There is no clearly defined Markdown standard, apart from the original writeup and implementation by John Gruber, which some consider <a href="/wiki/Abandonware" title="Abandonware">abandonware</a>.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17">[17]</a></sup><sup id="cite_ref-18" class="reference"><a href="#cite_note-18">[18]</a></sup> This has led to <a href="/wiki/Fragmentation_(programming)" class="mw-redirect" title="Fragmentation (programming)">fragmentation</a> as different vendors write their own variants of the language to correct flaws or add missing features.</p>
<p>A number of other <a href="/wiki/Lightweight_markup_language" title="Lightweight markup language">lightweight markup languages</a> extend Markdown by adding features (such as tables, footnotes, definition lists, and Markdown inside HTML blocks) not available in the original Markdown syntax. Among these are <a href="#Markdown_Extra">Markdown Extra</a>, <a href="/wiki/MultiMarkdown" title="MultiMarkdown">MultiMarkdown</a>, and the <a href="/wiki/Pandoc" title="Pandoc">Pandoc</a> Markdown extension.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">[19]</a></sup></p>
<p>The original Markdown implementation interprets constructs like <code>my_long_variable</code> as a request to emphasize "long". Many users found this confusing, so many later implementations such as PHP Markdown and Python Markdown do not implement middle word emphasis with underscores.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20">[20]</a></sup></p>
<p>Babelmark2<sup id="cite_ref-21" class="reference"><a href="#cite_note-21">[21]</a></sup> is a tool available to "[compare] the output of various implementations" to "promote discussion of how and whether certain vague aspects of the markdown spec should be clarified."<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">[22]</a></sup> Gruber has argued that complete standardisation would be mistaken: "Different sites (and people) have different needs. No one syntax would make all happy."<sup id="cite_ref-23" class="reference"><a href="#cite_note-23">[23]</a></sup></p>
<p>In March 2016 <a class="external mw-magiclink-rfc" rel="nofollow" href="//tools.ietf.org/html/rfc7763">RFC 7763</a> and <a class="external mw-magiclink-rfc" rel="nofollow" href="//tools.ietf.org/html/rfc7764">RFC 7764</a> were published. RFC 7763 introduced <a href="/wiki/MIME" title="MIME">MIME</a> type <small><code class="mw-highlight" id="" style="" dir="ltr">text/markdown</code></small> with the original variant. RFC&#160;7764 discusses and registered the variants <a href="/wiki/MultiMarkdown" title="MultiMarkdown">MultiMarkdown</a>, GitHub Flavored Markdown (GFM), <a href="/wiki/Pandoc" title="Pandoc">Pandoc</a>, <a href="#CommonMark">CommonMark</a>, and Markdown Extra among others.<sup id="cite_ref-IANA_24-0" class="reference"><a href="#cite_note-IANA-24">[24]</a></sup></p>
<h3><span class="mw-headline" id="CommonMark">CommonMark</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Markdown&amp;action=edit&amp;section=3" title="Edit section: CommonMark">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>From 2012, a group of people including <a href="/wiki/Jeff_Atwood" title="Jeff Atwood">Jeff Atwood</a> launched what Atwood characterized as a standardization effort.<sup id="cite_ref-FutureOfMarkdown_25-0" class="reference"><a href="#cite_note-FutureOfMarkdown-25">[25]</a></sup> A community website now aims to "document various tools and resources available to document authors and developers, as well as implementors of the various markdown implementations".<sup id="cite_ref-26" class="reference"><a href="#cite_note-26">[26]</a></sup> In September 2014, Gruber objected to the usage of "Markdown" in the name of this effort and it was rebranded as a new dialect named CommonMark.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27">[27]</a></sup><sup id="cite_ref-28" class="reference"><a href="#cite_note-28">[28]</a></sup> CommonMark.org published several versions of a specification, reference implementation, and test suite, and "[plans] to announce a finalized 1.0 spec and test suite in 2017."<sup id="cite_ref-commonmark.org_29-0" class="reference"><a href="#cite_note-commonmark.org-29">[29]</a></sup></p>
<p>In 2017 GitHub released a formal specification of their GitHub Flavored Markdown (GFM) that is based on CommonMark.<sup id="cite_ref-30" class="reference"><a href="#cite_note-30">[30]</a></sup> It follows the CommonMark specification exactly except for <i>tables, strikethrough, autolinks and task lists,</i> which the GitHub spec has added as extensions<sup id="cite_ref-31" class="reference"><a href="#cite_note-31">[31]</a></sup><i>.</i></p>
<h3><span class="mw-headline" id="Markdown_Extra">Markdown Extra</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Markdown&amp;action=edit&amp;section=4" title="Edit section: Markdown Extra">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Markdown Extra is a <a href="/wiki/Lightweight_markup_language" title="Lightweight markup language">lightweight markup language</a> based on Markdown implemented in <a href="/wiki/PHP" title="PHP">PHP</a> (originally), <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> and <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>. It adds features not available with plain Markdown syntax. Markdown Extra is supported in <a href="/wiki/Content_management_system" title="Content management system">content management systems</a>, for example <a href="/wiki/Drupal" title="Drupal">Drupal</a>,<sup id="cite_ref-32" class="reference"><a href="#cite_note-32">[32]</a></sup> <a href="/wiki/TYPO3" title="TYPO3">TYPO3</a><sup id="cite_ref-33" class="reference"><a href="#cite_note-33">[33]</a></sup> and <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a>.<sup id="cite_ref-34" class="reference"><a href="#cite_note-34">[34]</a></sup></p>
<p>It adds the following features to Markdown:</p>
<ul>
<li>Markdown inside <a href="/wiki/HTML" title="HTML">HTML</a> blocks</li>
<li>Elements with id/class attribute</li>
<li>Fenced code blocks</li>
<li>Tables<sup id="cite_ref-35" class="reference"><a href="#cite_note-35">[35]</a></sup></li>
<li>Definition lists</li>
<li>Footnotes</li>
<li>Abbreviations</li>
</ul>

<h2><span class="mw-headline" id="Implementations">Implementations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Markdown&amp;action=edit&amp;section=6" title="Edit section: Implementations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Implementations of Markdown are available for over a dozen programming languages; in addition, many platforms and frameworks support Markdown.<sup id="cite_ref-36" class="reference"><a href="#cite_note-36">[36]</a></sup> For example, Markdown plugins exist for every major blogging platform.<sup id="cite_ref-37" class="reference"><a href="#cite_note-37">[37]</a></sup></p>
<p>While Markdown is a minimal markup language and is easily read and edited with a normal text editor, there are specially designed editors that preview the files with styles, which are available for all major platforms. Many general purpose text and code editors have syntax highlighting plugins for Markdown built into them or available as optional download. Editors may feature a side-by-side preview window or render the code directly in a <a href="/wiki/WYSIWYG" title="WYSIWYG">WYSIWYG</a> fashion.</p>
<ul>
<li>The sourcecode documentation generator <a href="/wiki/Doxygen" title="Doxygen">Doxygen</a> supports Markdown with extra features.<sup id="cite_ref-38" class="reference"><a href="#cite_note-38">[38]</a></sup></li>
<li><a href="/wiki/RStudio" title="RStudio">RStudio</a>, an <a href="/wiki/Integrated_development_environment" title="Integrated development environment">IDE</a> for <a href="/wiki/R_(programming_language)" title="R (programming language)">R</a> provides a <a href="/wiki/C%2B%2B" title="C++">C++</a> <a href="/wiki/Wrapper_function" title="Wrapper function">wrapper function</a> for a markdown variant called sundown.<sup id="cite_ref-39" class="reference"><a href="#cite_note-39">[39]</a></sup></li>
<li>GitHub Flavored Markdown (GFM) treats newlines in paragraph-like content as real line breaks, ignores underscores in words, and adds <a href="/wiki/Syntax_highlighting" title="Syntax highlighting">syntax highlighting</a>, <a href="/wiki/Task_list" class="mw-redirect" title="Task list">task lists</a>,<sup id="cite_ref-40" class="reference"><a href="#cite_note-40">[40]</a></sup> and tables.<sup id="cite_ref-gfm_on_github2_41-0" class="reference"><a href="#cite_note-gfm_on_github2-41">[41]</a></sup></li>
<li>Discount - a <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> implementation.<sup id="cite_ref-42" class="reference"><a href="#cite_note-42">[42]</a></sup><sup id="cite_ref-43" class="reference"><a href="#cite_note-43">[43]</a></sup></li>
<li>MarkAPL is a converter written in Dyalog <a href="/wiki/APL_(programming_language)" title="APL (programming language)">APL</a>. It supports fenced blocks, smart typography, link references, and special attributes, and can generate a table of contents.<sup id="cite_ref-44" class="reference"><a href="#cite_note-44">[44]</a></sup></li>
<li>PHP Markdown - a library package that includes the PHP Markdown parser and its sibling PHP Markdown Extra with additional features.<sup id="cite_ref-45" class="reference"><a href="#cite_note-45">[45]</a></sup></li>
<li><a rel="nofollow" class="external text" href="https://github.com/lunet-io/markdig">Markdig</a> - A .NET library which follows the <a rel="nofollow" class="external text" href="http://commonmark.org/">CommonMark</a> <a rel="nofollow" class="external text" href="http://spec.commonmark.org/">specifications</a>, and has a large collection of extensions with the ability for the user to create their own.</li>
</ul>
